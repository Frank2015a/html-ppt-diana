# 商用演示设计原则

本文档定义了创建专业商用 HTML 演示文稿的核心设计原则和最佳实践。

## 核心设计原则

### 1. 专业性至上

**禁止使用元素：**
- 表情符号 (emoji) - 降低专业感
- 过度装饰性字体 - 影响可读性
- 低分辨率图片 - 损害品牌形象
- 动画 GIF - 分散注意力

**推荐使用：**
- 简洁的无衬线字体（如 PingFang SC, Segoe UI）
- 高质量矢量图标
- 专业摄影作品或插图
- 品牌配色方案

### 2. 视觉一致性

**统一的布局元素：**
- 顶部标题区域保持一致的高度和样式
- 相同类型的页面使用相同的布局模式
- 统一的字体大小层级（标题、正文、注释）
- 一致的间距和边距

**色彩系统：**
- 主色：用于标题、重点强调 (Primary Color)
- 辅助色：用于链接、按钮、图表 (Secondary Color)
- 强调色：用于警告、重要信息 (Accent Color)
- 中性色：用于正文、背景 (Neutral Colors)

### 3. 内容层次

**F 型阅读模式：**
- 重要信息放在左上角
- 标题在页面顶部
- 关键数据使用视觉突出

**字体层级：**
- 一级标题：48-64px (封面)
- 二级标题：28-36px (页面标题)
- 三级标题：24-28px (段落标题)
- 正文：20-24px
- 注释：16-18px

### 4. 留白原则

**有效的留白：**
- 每页边距至少 60-80px
- 元素之间保持足够间距
- 避免内容过于拥挤
- 使用网格系统对齐元素

### 5. 数据可视化

**图表选择：**
- 柱状图：比较数值大小
- 折线图：展示趋势变化
- 饼图：展示占比（限 5 个以内分类）
- 散点图：展示相关性
- 雷达图：多维度评估

**图表设计原则：**
- 清晰的标题和坐标轴标签
- 使用品牌色系
- 避免 3D 效果
- 提供数据标签
- 保持图表简洁

### 6. 媒体素材使用

**图片：**
- 分辨率至少 1920x1080
- 使用高质量商业摄影
- 统一的滤镜风格
- 适当裁剪以符合布局

**视频：**
- 控制在 2 分钟以内
- 提供静音选项
- 添加字幕说明
- 确保格式兼容性 (MP4/H.264)

### 7. 交互设计

**导航：**
- 明确的页码指示
- 可见的上一页/下一页按钮
- 进度条显示演示进度
- 支持键盘快捷键

**响应式：**
- 在不同屏幕尺寸下保持可读性
- 触摸友好的按钮尺寸
- 适配投影仪常见分辨率 (1920x1080, 1280x720)

## 商务演示特殊要求

### 客户演示

**必须包含：**
- 公司 Logo 和品牌元素
- 清晰的联系方式
- 案例研究和数据支撑
- 可信度证明（认证、奖项）

**避免：**
- 过多的技术细节
- 内部术语和缩写
- 未经验证的数据
- 竞争对手贬低

### 投资路演

**关键页面：**
1. 封面 - 公司名称和定位
2. 问题 - 市场痛点
3. 解决方案 - 产品/服务
4. 市场规模 - TAM/SAM/SOM
5. 商业模式 - 如何盈利
6. 竞争优势 - 护城河
7. 团队 - 核心成员
8. 财务预测 - 收入增长
9. 融资需求 - 金额和用途
10. 联系方式

### 培训课程

**教学设计：**
- 章节清晰划分
- 学习目标明确
- 案例和练习穿插
- 关键要点总结
- 资源链接提供

**视觉辅助：**
- 流程图展示步骤
- 对比表格
- 实际操作截图
- 代码示例（如适用）

## 可访问性考虑

### 颜色对比

- 文字与背景对比度至少 4.5:1
- 重要信息不仅依赖颜色区分
- 提供色盲友好的配色方案

### 字体可读性

- 最小字体 16px
- 行高至少 1.5
- 避免全大写长文本
- 充足的字间距

### 内容结构

- 使用语义化标题层级
- 提供替代文本 (alt text)
- 表格包含表头
- 链接文字描述清晰

## 检查清单

在发布演示文稿前，确认：

- [ ] 所有页面标题统一样式
- [ ] 没有使用 emoji 表情
- [ ] 图片清晰且高分辨率
- [ ] 视频可以正常播放
- [ ] 图表数据准确
- [ ] 文字无拼写错误
- [ ] 色彩搭配协调
- [ ] 留白适当不拥挤
- [ ] 键盘导航功能正常
- [ ] 在目标设备上测试过
- [ ] 联系方式准确无误
- [ ] 品牌元素正确展示
